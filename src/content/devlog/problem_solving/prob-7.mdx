외부 연동 API 호출 시 응답 대기 시간 동안 스레드가 블로킹되어 전체 애플리케이션의 처리량(Throughput)이 급감하는 성능 저하가 발생했습니다.

## 1. Blocking I/O의 한계점 발견

TPS가 증가함에 따라 WAS의 스레드 풀이 모두 API 응답 대기 상태(Wait)로 전이되어 정작 필요한 로직을 처리하지 못하는 '스레드 고갈' 현상이 확인되었습니다.

## 2. Non-blocking 비동기 응답 처리 고도화

기존 RestTemplate 대신 WebClient를 도입하고, CompletableFuture와 리액티브 스트림 구조를 활용하여 비동기 논블로킹 방식으로 응답 처리 로직을 전면 재작성했습니다.

동일 리소스 하에서 동시 처리 가능 요청 수가 약 3배 증가했습니다. 리액티브 프로그래밍 패러다임을 팀 내에 전파하고 모니터링 환경을 구축하는 것이 다음 목표입니다.

분산 환경에서 데이터 정합성을 유지하기 위해 Redis 기반의 분산 락을 구현한 경험을 공유합니다.

## 1. 문제 상황
다수의 인스턴스가 동시에 동일한 데이터(예: 재고)에 접근하여 수정할 때 발생하는 Race Condition으로 인해 데이터 정합성이 깨지는 문제가 발생했습니다. 기존의 DB Lock만으로는 성능 저하와 Deadlock 위험이 있었습니다.

## 2. 해결 방법
Redis를 활용한 분산 락(Distributed Lock)을 도입했습니다. 특히 단일 Redis 노드의 장애를 대비하여 Redlock 알고리즘 기반의 Redisson 라이브러리를 활용했습니다.

*   **Lock 획득 및 해제:** 타임아웃을 설정하여 특정 노드가 죽더라도 Lock이 영구적으로 점유되는 현상을 방지했습니다.
*   **스핀 락(Spin Lock) 회피:** Redisson의 Pub/Sub 기능을 활용하여 Lock 획득 대기 상태의 쓰레드들이 불필요하게 Redis에 요청을 보내는 부하를 줄였습니다.

## 3. 결과
동시성 보장을 통해 데이터 정합성 문제를 100% 해결했으며, DB에 가해지던 락 경합 부하를 Redis로 분산시켜 전체 트랜잭션 처리 속도도 향상되었습니다.

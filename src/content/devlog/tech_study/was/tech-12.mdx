웹 서버와 WAS(Web Application Server)는 백엔드 인프라의 최전방을 담당하는 핵심 컴포넌트입니다. 본 문서에서는 네 가지 대표적인 솔루션(Apache httpd, Nginx, Apache Tomcat, JEUS)의 특징과 장단점을 명확히 비교하고, 왜 시장에서 톰캣(Tomcat)이 독보적인 지위를 차지하고 있는지 분석합니다.

---

## 1. 정적 웹 서버 (Web Server) 군단

<Indent>
웹 서버는 주로 HTML, CSS, 이미지 같은 정적 에셋(Static Asset)을 빠르게 서빙하고, 리버스 프록시(Reverse Proxy) 역할을 수행합니다.

### ① Apache httpd (전통의 강자)
*   **👍 장점:** 
    *   **수많은 모듈과 기능:** 수십 년간 축적된 확장 모듈(mod_proxy, mod_ssl 등)이 있어 설정만으로 거의 모든 기능을 구현할 수 있습니다.
    *   **디렉터리별 세부 제어:** `.htaccess` 파일을 통해 서버 재시작 없이 폴더별로 권한이나 설정을 유연하게 바꿀 수 있습니다.
*   **👎 단점:** 
    *   **클라이언트당 스레드 배정 (Process/Thread-per-connection):** 접속자 수가 폭발하면 스레드가 비례해서 늘어납니다. 이로 인해 메모리 소모가 극심해지고 문맥 교환(Context Switching) 비용이 커져 대규모 동시 접속(C10K 문제)에 취약합니다.

### ② Nginx (차세대 트렌드)
*   **👍 장점:** 
    *   **이벤트 기반 아키텍처 (Event-Driven):** 고정된 적은 수의 프로세스가 수많은 요청을 비동기 이벤트로 처리합니다. CPU와 메모리 소모량이 Apache httpd에 비해 압도적으로 적습니다.
    *   **초고속 정적 서빙 및 프록시:** 동시 접속자가 수만 명으로 몰려도 지연이 거의 없으며, 로드 밸런싱 및 리버스 프록시 성능이 세계 최고 수준입니다.
*   **👎 단점:** 
    *   동적 모듈 로딩이 까다롭고, Apache httpd의 `.htaccess`처럼 개발자가 서버 설정을 실시간으로 파인 튜닝하기 어렵습니다.
</Indent>

## 2. 동적 웹 애플리케이션 서버 (WAS) 군단

<Indent>
WAS는 자바(Java) 코드를 구동하여 DB를 조회하는 등 동적 비즈니스 로직을 처리합니다.

### ③ Apache Tomcat (범용 오픈소스 WAS)
*   **👍 장점:** 
    *   **가볍고 빠름:** 자바의 웹 표준 규격 중 웹 핵심 기능인 서블릿 컨테이너(Servlet Container - JSP, 서블릿)만 구현했기 때문에 매우 가볍고 메모리 탑재량이 적습니다.
    *   **방대한 커뮤니티:** 전 세계 거의 모든 자바 개발자가 사용하므로 레퍼런스가 무한대에 가깝습니다.
*   **👎 단점:** 
    *   과거 완전한 자바 표준 엔터프라이즈 사양(전체 Jakarta EE / 구 Java EE)을 다 지원하지 못해, 대규모 분산 트랜잭션(EJB 등) 기능이 필요한 경우 외부 라이브러리를 추가해야 했습니다. (최근에는 Tomcat 정식 버전도 많은 표준을 흡수함)

### ④ JEUS (국산 엔터프라이즈 WAS)
*   **👍 장점:** 
    *   **Full Spec 규격 지원:** Jakarta EE 표준 스펙을 100% 만족하는 중대형 WAS입니다.
    *   **대기업/금융권 최적화 클러스터링:** 세션 클러스터링 및 분산 트랜잭션(2PC) 관리가 매우 정교하여 대규모 트래픽 환경에서 데이터 유실 없는 안정성을 보장합니다. 국내 기술 지원(티맥스소프트)이 신속합니다.
*   **👎 단점:** 
    *   **고가의 라이선스 비용:** 오픈소스가 아닌 상용 소프트웨어이므로 도입 및 유지보수 비용이 매우 비쌉니다. 무겁고 초기 구동 속도가 느려 개인 프로젝트나 스타트업에는 적합하지 않습니다.
</Indent>

## 4. 시장에서 Tomcat을 압도적으로 많이 사용하는 이유

<Indent>
국내외를 막론하고 신규 프로젝트나 서비스 아키텍처를 볼 때 **"Nginx(앞단) + Tomcat(뒷단)"** 조합은 사실상 표준(De-facto standard)으로 통합니다. 톰캣의 독점적인 인기 비결은 다음과 같습니다.

### 1) Spring Boot의 심장 (내장 WAS 트렌드)
현재 전 세계 자바 백엔드 시장을 지배하고 있는 Spring Boot는 빌드 시 톰캣을 내장(Embedded Tomcat) 상태로 패키징합니다. 개발자가 별도의 WAS 서버를 설치하고 설정할 필요 없이, 그저 자바 파일(`.jar`) 하나만 실행하면 즉시 서버가 구동되는 압도적인 개발 편의성을 제공합니다.

### 2) "적당함"의 미학 (라이트한 스펙)
과거(2000년대 초반)에는 거대한 비즈니스를 하려면 EJB 같은 기술이 필수적이어서 JEUS나 WebLogic 같은 무겁고 비싼 상용 WAS가 강제되었습니다. 하지만 POJO 중심의 Spring 프레임워크가 주류가 되면서 무거운 엔터프라이즈 기능들이 필요 없어졌습니다. 스프링의 핵심 기능을 돌리기에는 서블릿 규격만 꽉 쥐고 있는 가벼운 톰캣이 성능적으로나 비용적으로 가장 최적의 선택지였습니다.

### 3) 비용 절감과 클라우드(MSA) 환경과의 궁합
클라우드 인프라(AWS 등)와 컨테이너 환경(Docker, Kubernetes)에서는 인스턴스를 수시로 껐다 켜고 확장(Scale-out)해야 합니다.
JEUS 같은 무거운 WAS는 대당 라이선스 비용이 청구되어 클라우드 환경에서 대수를 늘릴 때 비용 폭탄을 맞으며, 컨테이너가 켜지는 속도마저 무겁고 느립니다.
반면 **Tomcat은 라이선스 비용이 0원이며 메모리를 적게 먹고 가벼워**, 컨테이너 기반의 마이크로서비스 아키텍처(MSA) 환경에 무한히 복제·배포하기에 가장 최적화되어 있습니다.
</Indent>

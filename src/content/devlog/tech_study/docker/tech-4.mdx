Docker를 실무에 적용하기 위해 반드시 알아야 할 주요 명령어와 옵션들을 상세하게 살펴봅니다.

---

## 1. 컨테이너 실행 (`docker run`)

<Indent>
컨테이너를 생성하고 시작하는 가장 핵심적인 명령어입니다. 옵션을 조합하여 컨테이너의 환경을 완벽히 제어할 수 있습니다.

```bash
docker run -d -p 8080:80 --name my_webserver -e ENV_VAR=value nginx:latest
```

*   `-d` (Detached): 컨테이너를 백그라운드 모드로 실행합니다. (주로 웹서버, DB 등에 필수)
*   `-p` (Publish): 포트 포워딩을 설정합니다. `[호스트 포트]:[컨테이너 포트]` 형태로 작성합니다.
*   `--name`: 실행할 컨테이너에 사용자가 식별하기 쉬운 이름을 부여합니다. 미지정 시 무작위 문자열이 생성됩니다.
*   `-e` (Environment): 컨테이너 내부에서 사용할 환경변수를 설정합니다.
*   `-v` (Volume): 볼륨 마운트를 설정합니다. (스토리지 섹션 참조)
</Indent>

---

## 2. 상태 및 로그 확인

<Indent>
실행 중인 서비스의 상태를 파악하거나 디버깅을 수행할 때 사용합니다.

*   `docker ps`: 현재 실행 중인 컨테이너 목록만 출력합니다.
*   `docker ps -a`: 종료되거나 실패한 컨테이너를 포함한 모든 목록을 출력합니다.
*   `docker logs [container_name]`: 컨테이너 내부의 로그를 출력합니다.
    *   `-f` 옵션을 추가하면 `tail -f`처럼 실시간 로그를 추적할 수 있습니다.
*   `docker inspect [container_name]`: 컨테이너의 IP 주소, 마운트된 볼륨, 환경변수 등 아주 상세한 메타데이터를 JSON 형태로 확인합니다.
</Indent>

---

## 3. 컨테이너 내부 접근 및 제어

<Indent>
가동 중인 컨테이너 내부에 직접 명령을 내리거나 쉘(Shell)을 얻기 위해 사용합니다.

*   `docker exec -it [container_name] /bin/bash`
    *   컨테이너 내부에 Bash Shell을 열어 접속합니다. (알파인 리눅스 기반일 경우 `/bin/sh` 사용)
    *   `-i` (Interactive)와 `-t` (TTY) 옵션이 필수적입니다.
*   `docker stop [container_name]`: 컨테이너의 프로세스에 SIGTERM을 보내 안전하게(Graceful) 종료합니다.
*   `docker rm -f [container_name]`: 실행 여부와 관계없이 강제로 컨테이너를 삭제합니다.
</Indent>

---

## 4. 리소스 정리 (`prune`)

<Indent>
개발을 반복하다 보면 찌꺼기 이미지와 중지된 컨테이너가 디스크 용량을 크게 차지하게 됩니다.

*   `docker system prune`: 중지된 컨테이너, 사용되지 않는 네트워크, 이름 없는(Dangling) 이미지 등을 일괄 삭제합니다.
*   `docker system prune -a`: 위 항목에 더해 현재 어떤 컨테이너도 사용하지 않는 '모든' 이미지를 삭제합니다. (공간 확보에 유용하지만, 다음 빌드 시 새로 이미지를 받아와야 합니다.)
</Indent>
